# 향후 콘텐츠 아이디어 기록

> 이 문서는 프로토타입 단계에서 검토했지만 보류한 콘텐츠 아이디어를 기록합니다.
> 본개발(Unity) 단계에서 참고할 수 있습니다.

---

## 논의 배경

**질문**: Slay the Spire는 캐릭터 선택 후 지도를 보여줘서 전략을 세우게 유도하는데, 우리 게임은 캐릭터 선택 이후 카드 배틀 전에 어떤 콘텐츠를 넣으면 좋을까?

**결론**: 프로토타입 단계에서는 **추가 콘텐츠 없이** 현재 시스템 유지

---

## 현재 시스템 분석

### 캐릭터 초기화 씬 (이미 충분히 풍부함)

| 단계 | 내용 | 판단 깊이 |
|------|------|----------|
| 캐릭터 선택 | 전사/법사/도적 중 1개 | 플레이 스타일 결정 |
| 특성 분배 | 5점을 5개 특성에 분배 | 빌드 방향 결정 |
| 조합 보너스 | 특정 조합 시 시너지 카드 | 최적화 고민 |

**현재 시스템의 장점**:
- 이미 깊은 Layer 2 판단 제공
- 랜덤 요소 없음 → 리셋 어뷰징 불가
- 선택이 런 전체에 영향

---

## 검토한 옵션들 (보류)

### 옵션 1: 유물 갈림길 (Relic Fork)

**개요**: 3개 유물 중 1개 선택, 나머지 영구 소멸. 선택한 유물은 런 전체에 효과 지속.

**예시**:
```
[A] 피의 목걸이: 매 턴 HP -1, 카드 +1 드로우
[B] 강철 방패: 전투 시작 방어 +5, 에너지 -1 (영구)
[C] 행운의 동전: 전투 종료 시 50% 보상 2배 / 50% 없음
```

**플레이어 감정**: 전략적 분석 → 포기의 아쉬움 → 빌드 완성감

**보류 이유**: 현재 특성 시스템과 역할 중복

---

### 옵션 2: 축복 거래 (Blessing Trade)

**개요**: 강력한 축복 + 영구 저주 패키지. 수락하면 런 전체에 적용.

**예시**:
```
[거래 1] 광전사의 힘
- 축복: 매 턴 에너지 +1
- 저주: 턴 종료 시 HP -3

[거래 2] 현자의 눈
- 축복: 적 다음 행동 표시
- 저주: 카드 드로우 -1
```

**플레이어 감정**: 리스크 계산 → 도박 심리 → 극단적 빌드

**보류 이유**: 복잡도 증가, 하이리스크 밸런싱 어려움

---

### 옵션 3: 덱 변형 제단 (Deck Altar)

**개요**: 덱 자체를 영구적으로 변형 (카드 제거/복제/변환)

**예시**:
```
[A] 정화: 카드 2장 영구 제거 → 희귀 카드 1장 획득
[B] 복제: 카드 1장 선택 → 해당 카드 3장으로 복제
[C] 변환: 카드 1장 → 같은 비용의 랜덤 카드로 변환
```

**플레이어 감정**: 덱 분석 → 영구 결정 불안 → 빌드 집중

**보류 이유**: 현재 특성 카드 시스템과 중복

---

### 옵션 4: 실시간 갈림길 (Time-Pressured Fork)

**개요**: 10초 내 3개 갈림길 중 1개 선택. 시간 내 선택 안 하면 랜덤 배정.

**보류 이유**:
- PRD "사고할 수 있는 선택" 원칙과 충돌
- 시간 압박 → "사고"보다 "반사"에 가까움

---

## 추가 콘텐츠 설계 시 고려사항

### 필수 요건

| 요건 | 설명 |
|------|------|
| **사고 시간 보장** | 시간 제한 없이 충분히 고민 가능해야 함 |
| **지속적 영향** | 선택이 해당 배틀뿐 아니라 런 전체에 영향 |
| **리셋 방지** | 상황적 가치 설계로 "정답" 없앰 |
| **PRD 정합성** | 가치 충돌 + 판단 누적 + 재평가 |

### 리셋 어뷰징 방지 설계

**문제**: 랜덤 선택지 → "좋은 거 나올 때까지 재시작"

**해결책: 상황적 가치 설계**
- 어떤 선택지도 상황에 따라 가치가 달라짐
- 빌드 의존적 가치: 현재 덱/유물에 따라 최선이 달라짐
- 고정 시퀀스: 캐릭터에 따라 선택지 순서 고정

**예시**:
```
유물 A: 드로우 덱에 강력 ⭐⭐⭐⭐⭐ / 에너지 덱에 약함 ⭐
유물 B: 에너지 덱에 강력 ⭐⭐⭐⭐⭐ / 드로우 덱에 약함 ⭐

→ 리셋해도 "더 좋은 조합"이 아니라 "다른 조합"만 나옴
```

## 향후 확장 시 참고

본개발에서 추가 콘텐츠가 필요할 경우:

| 콘텐츠 | 위치 | 연계 시스템 |
|--------|------|------------|
| 유물 시스템 | 배틀 보상 | 배틀 사이 선택 |
| 이벤트 시스템 | 런 중 랜덤 | 스토리/선택 분기 |
| 상점 시스템 | 배틀 후 | 협상가 특성 연계 |
| 지도 시스템 | 런 전체 | 경로 선택 (StS 스타일) |

## 최종 결정

```
프로토타입 게임 플로우:
메인메뉴 → 캐릭터 초기화 (캐릭터+특성 선택) → 배틀 → 결과

추가 콘텐츠: 없음
```

**이유**:
1. 현재 캐릭터 초기화 시스템이 이미 충분히 풍부
2. 프로토타입 목적은 핵심 메카닉 검증 (카드 선택의 가치 충돌)
3. 추가 콘텐츠는 본개발(Unity) 단계에서 검토
