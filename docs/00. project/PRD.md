## 0. Game Pillar (헌법 문장)

이 게임은 플레이어가  
불완전한 정보 속에서 여러 가치 축이 동시에 충돌하는 상황에 즉각 반응하며,  
그 순간적인 판단들이 누적되어  
중간 무게의 결정을 다시 요구받는 게임이다.

---

## 1. Core Experience (핵심 경험)

플레이어는 게임 내내 다음과 같은 상태를 유지해야 한다.

- 선택은 멈추고 충분히 고민한 뒤 내리는 결정이 아니다
- 상황이 먼저 발생하고, 플레이어는 즉각 반응한다
- 그 순간에는 최선이라고 느낀 선택이
  이후 돌아보면 다른 가치를 희생했음을 깨닫게 된다
- 이 누적된 선택의 결과가
  다시 중간 무게의 판단을 요구한다

몰입의 원천은  
**반응 → 누적 → 재평가**의 반복에서 발생한다.

---

## 2. Value Conflict Model (가치 충돌 모델)

이 게임의 모든 주요 선택은  
서로 다른 가치 축이 동시에 충돌하는 형태로 제시된다.

### 고정된 가치 축

- 단기 이득 ↔ 장기 성장
- 안정성 ↔ 변동성
- 효율 ↔ 확장성
- 통제 ↔ 자유

### 설계 원칙

- 하나의 선택이 특정 가치 축에서 유리하다면,
  다른 축에서는 반드시 불리해야 한다
- 모든 축에서 유리한 선택이 발생하는 순간,
  해당 설계는 실패로 간주한다
- 상황에 따라 우선해야 할 가치가 달라져야 하며,
  항상 옳은 선택은 존재하지 않는다

---

## 3. Judgment Layer Model (판단 계층 구조)

이 게임의 판단은 두 개의 계층으로 구성된다.

### Layer 1: 반응 판단 (Reactive Micro Judgment)

- 빈도: 매우 잦음
- 무게: 가벼움
- 특징:
  - 상황이 먼저 제시되고 즉각 반응해야 한다
  - 제한된 정보만 제공된다
  - 결과는 빠르게 나타나지만 불완전하다
  - 실수 가능성이 항상 존재한다

목적:
- 플레이어의 가치 성향을 자연스럽게 누적시키는 것

---

### Layer 2: 중간 판단 (Reflective Mid Judgment)

- 빈도: 간헐적
- 무게: 중간
- 특징:
  - 여러 반응 판단의 결과가 반영된다
  - 복수의 가치 축이 동시에 명시된다
  - 일부 결과는 되돌리기 어렵다

목적:
- 플레이어에게
  “지금까지의 반응 방향이 옳았는가?”
  를 다시 묻는 것

---

## 4. Core Loop (판단 루프)

1. 반응 기반 미시 판단이 연속적으로 발생한다
2. 선택의 결과로 자원, 상태, 가치 편향이 누적된다
3. 특정 조건에 도달하면 중간 판단이 트리거된다
4. 플레이어는 다음 중 하나를 선택한다:
   - 지금까지의 판단 방향을 강화
   - 일부 방향을 수정
   - 자원을 사용해 복합 선택(A+B) 시도
5. 판단 지형이 재편성된다
6. 다시 반응 판단 루프로 복귀한다

중간 판단은 휴식 구간이 아니라,  
반응 판단을 다시 긴장 상태로 되돌리는 장치다.

---

## 5. Resource Philosophy (자원 철학)

자원은 이 게임에서 다음 역할을 가진다.

- 판단을 제거하지 않는다
- 판단의 범위와 깊이를 확장한다

### 반응 판단에서의 자원
- 실수의 여지를 약간 완화한다
- 선택의 여백을 제공한다
- 반응 자체를 자동화하지 않는다

### 중간 판단에서의 자원
- 복합 선택(A+B)을 가능하게 한다
- 판단 결과의 강도와 범위를 조절한다

자원 축적은  
고민의 종료가 아니라 고민의 난이도 상승을 의미한다.

---

## 6. Design Constraints (설계 제약)

### 반드시 지켜야 할 것

- 모든 주요 선택은 최소 2개 이상의 가치 축을 포함해야 한다
- 자원이 많아질수록 고민 시간이 줄어들면 안 된다
- 복합 선택(A+B) 이후에도 즉시 새로운 판단 상황이 등장해야 한다
- 반응 판단은 빠르게 유지되어야 한다
- 중간 판단에서는 반드시 멈칫이 발생해야 한다

### 절대 금지

- 단일 수치로 판단이 끝나는 구조
- 특정 가치 축이 항상 우선되는 메타
- 복합 선택(A+B)이 명백한 정답이 되는 밸런스
- 반응 판단이 의미 없는 습관 입력으로 전락하는 상황

---

## 7. Core Fun Hypothesis (핵심 재미 가설)

플레이어는  
즉각 반응해야 하는 선택 상황에서  
순간적으로 내린 판단이 누적되어  
이후 더 무거운 판단으로 되돌아올 때,  
자원이 충분한 상태에서도  
즉시 결정을 내리지 못하고  
비교, 재고, 우선순위 설정을 반복한다면  
이 게임의 핵심 재미는 유지된다.

---

## 8. Failure Definition (Kill Criteria)

다음 중 하나라도 발생하면
핵심 재미는 붕괴된 것으로 간주한다.

- 반응 판단이 무의미한 반사 입력이 된다
- 중간 판단에서 항상 정답처럼 보이는 선택이 고정된다
- 자원 축적이 고민 시간을 급격히 감소시킨다
- 플레이어가 자신의 과거 판단을 돌아볼 이유를 느끼지 못한다
