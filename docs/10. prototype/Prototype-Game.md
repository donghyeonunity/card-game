# Phase 1 프로토타입 검증 문서 (v2)

## 1. 검증 목표

### 핵심 가설
> 플레이어가 A/B 선택에서 **가치 충돌을 명확히 인지**하는가?

### 검증 범위
- **검증할 것**: "읽고 멈추는가", "가치 충돌을 인지하는가"
- **검증하지 않을 것**: 연출, 사운드, 난이도 밸런스, 완성도

### Phase 1 경험 정의
- 시간 제한 없음 (충분히 읽고 고민)
- 적과의 대치 상황에서 카드 선택
- 각 선택에서 최소 2개 이상의 가치 축이 충돌해야 함

---

## 2. 전투 시스템 (신규)

### 2.1 목적
카드 선택에 **맥락과 긴장감**을 부여하여 가치 충돌이 체감되게 함.

### 2.2 유닛 스펙
| 요소 | 값 | 비고 |
|------|-----|------|
| 적 HP | 100 | 처치 목표 |
| 적 반격 | 10/턴 | 매 턴 플레이어에게 데미지 |
| 플레이어 HP | 50 | 생존 조건 |
| 시작 에너지 | 0 | 카드로 획득 |

### 2.3 승리/패배 조건
- **승리**: 적 HP를 0으로 만듦
- **패배**: 플레이어 HP가 0이 됨
- **암묵적 턴 제한**: 5턴 내 처치 필요 (반격 10 × 5 = 50)

### 2.4 게임 흐름
```
1. 게임 시작
   ├── 적 등장 (HP 100)
   └── 플레이어 상태 (HP 50, 에너지 0)

2. 턴 루프 ─────────────────────────────
   │
   ├── [카드 제시] A/B 선택
   │       ↓
   ├── [선택 결과] 에너지/효과 적용
   │       ↓
   ├── [공격 단계] 에너지 소모 → 적 HP 감소
   │       ↓
   └── [적 반격] 플레이어 HP -10

3. 종료 판정
   ├── 적 HP ≤ 0 → 승리
   └── 플레이어 HP ≤ 0 → 패배
```

### 2.5 간접 연결 구조
카드는 직접 데미지가 아닌 **자원(에너지)**을 통해 적과 연결됨.

```
[카드 선택] → [에너지 획득/효과]
                    ↓
            [공격 행동] 에너지 3 → 데미지 20
                    ↓
              [적 HP 감소]
```

**핵심 트레이드오프**: "지금 공격할까 vs 에너지 더 모을까"

## 3. 설계 원칙

### 3.1 카드 추상도: 중간 수준
| 수준 | 예시 | 적합성 |
|------|------|--------|
| 추상적 | "안정성 +2" vs "변동성 +3" | X |
| **중간** | "에너지 3 즉시" vs "2턴간 2씩" | O - 채택 |
| 구체적 | "화염구 발사" vs "얼음 방패" | X |

### 3.2 가치축 표시: 혼합 방식
- **태그 표시 카드**: [단기↑ 장기↓] 형태로 명시
- **텍스트만 카드**: 텍스트에서 가치 충돌 추론

### 3.3 공격 메커니즘
- **기본 공격**: 에너지 3 소모 → 데미지 20
- 카드 효과로 공격력 강화/변형 가능
- 에너지 부족 시 공격 불가 (다음 턴으로)

---

## 4. 가치 충돌 모델

### 4개 고정 가치축
| 축 | 전투 맥락에서의 의미 |
|----|----------------------|
| **단기 ↔ 장기** | 즉시 에너지/공격 vs 턴 투자 후 큰 효과 |
| **안정 ↔ 변동** | 확정 데미지 vs 무작위 고위험 고보상 |
| **효율 ↔ 확장** | 자원 최적화 vs 선택지 확보 |
| **통제 ↔ 자유** | 행동 제한 + 안전 vs 위험 감수 + 자유 |

## 5. 예시 카드 세트 (6장)

### 카드 1: 에너지 수급 (단기↔장기 | 태그 O)
```
┌─────────────────────────────────────┐
│  [1] 에너지 수급                     │
├─────────────────────────────────────┤
│  A: 급속 충전                        │
│     - 에너지 +3 즉시 획득            │
│     - 이번 턴 공격 가능              │
│                                     │
│  ─────────────────────────────────  │
│                                     │
│  B: 효율 충전                        │
│     - 이번 턴 에너지 +0              │
│     - 다음 2턴간 에너지 +2씩 (총 4)   │
├─────────────────────────────────────┤
│  [태그: 단기↑ / 장기↑]               │
└─────────────────────────────────────┘
```

### 카드 2: 공격 방식 (안정↔변동 | 태그 O)
```
┌─────────────────────────────────────┐
│  [2] 공격 방식                       │
├─────────────────────────────────────┤
│  A: 정밀 타격                        │
│     - 에너지 3 소모                  │
│     - 확정 20 데미지                 │
│                                     │
│  ─────────────────────────────────  │
│                                     │
│  B: 강타                             │
│     - 에너지 3 소모                  │
│     - 10~40 무작위 데미지            │
│     - 크리티컬 시 적 1턴 기절         │
├─────────────────────────────────────┤
│  [태그: 안정↑ / 변동↑]               │
└─────────────────────────────────────┘
```

### 카드 3: 자원 배분 (효율↔확장 | 태그 X)
```
┌─────────────────────────────────────┐
│  [3] 자원 배분                       │
├─────────────────────────────────────┤
│  A: 집중 투자                        │
│     - 보유 에너지 전부 소모           │
│     - 소모량 × 8 데미지              │
│     - (6 에너지 = 48 데미지)         │
│                                     │
│  ─────────────────────────────────  │
│                                     │
│  B: 분산 투자                        │
│     - 에너지 2 소모 → 10 데미지       │
│     - 남은 에너지 유지               │
│     - 다음 카드 선택 시 보너스 옵션    │
└─────────────────────────────────────┘
```

### 카드 4: 방어 선택 (통제↔자유 | 태그 X)
```
┌─────────────────────────────────────┐
│  [4] 방어 선택                       │
├─────────────────────────────────────┤
│  A: 선제 방어                        │
│     - 이번 턴 반격 무효화 (HP 유지)   │
│     - 다음 턴 공격만 가능 (카드 선택X) │
│                                     │
│  ─────────────────────────────────  │
│                                     │
│  B: 위험 감수                        │
│     - 반격 받음 (HP -10)             │
│     - 에너지 +2 보너스               │
│     - 다음 턴 모든 행동 가능          │
└─────────────────────────────────────┘
```

### 카드 5: 전략 전환 (단기↔장기 | 태그 O)
```
┌─────────────────────────────────────┐
│  [5] 전략 전환                       │
├─────────────────────────────────────┤
│  A: 즉시 회복                        │
│     - HP +15 회복                   │
│     - 이번 턴 공격 불가              │
│                                     │
│  ─────────────────────────────────  │
│                                     │
│  B: 공세 전환                        │
│     - 다음 공격 데미지 2배            │
│     - 반격 데미지도 2배 (HP -20)      │
├─────────────────────────────────────┤
│  [태그: 단기(생존)↑ / 장기(승리)↑]    │
└─────────────────────────────────────┘
```

### 카드 6: 최종 선택 (효율↔확장 | 태그 O)
```
┌─────────────────────────────────────┐
│  [6] 최종 선택                       │
├─────────────────────────────────────┤
│  A: 일점 집중                        │
│     - 보유 에너지 + HP 10 소모        │
│     - (에너지 + 10) × 5 데미지       │
│     - 모 아니면 도                   │
│                                     │
│  ─────────────────────────────────  │
│                                     │
│  B: 지속 공세                        │
│     - 에너지 2 소모 → 15 데미지       │
│     - 다음 턴 자동 에너지 +3          │
│     - 안정적 마무리                  │
├─────────────────────────────────────┤
│  [태그: 효율(리스크)↑ / 확장(안전)↑]  │
└─────────────────────────────────────┘
```

## 6. 카드 분배 요약

| 가치축 충돌 | 카드 번호 | 태그 표시 |
|-------------|-----------|-----------|
| 단기 ↔ 장기 | 1, 5 | 2태그 |
| 안정 ↔ 변동 | 2 | 1태그 |
| 효율 ↔ 확장 | 3, 6 | 1태그, 1텍스트 |
| 통제 ↔ 자유 | 4 | 텍스트만 |

**합계: 6장** (태그 4장, 텍스트만 2장)

## 7. 데이터 수집 항목

### 7.1 자동 수집 (시스템)
| 항목 | 설명 | 용도 |
|------|------|------|
| `decision_time_ms` | 카드 표시 ~ 선택까지 시간 | 멈춤 판단 |
| `choice` | A 또는 B | 비율 분석 |
| `hover_time_a_ms` | A 옵션 호버 시간 | 비교 행동 |
| `hover_time_b_ms` | B 옵션 호버 시간 | 비교 행동 |
| `switch_count` | A↔B 전환 횟수 | 고민 깊이 |
| `card_id` | 카드 식별자 | 카드별 분석 |
| `has_tag` | 태그 유무 | 혼합 방식 비교 |
| `turn_number` | 현재 턴 | 압박 상황 분석 |
| `player_hp` | 선택 시 플레이어 HP | 위기 상황 영향 |
| `enemy_hp` | 선택 시 적 HP | 승리 근접 영향 |

### 7.2 게임 결과
| 항목 | 설명 |
|------|------|
| `result` | 승리 / 패배 |
| `total_turns` | 총 턴 수 |
| `remaining_hp` | 종료 시 플레이어 HP |

### 7.3 설문 (종료 후)
| 문항 | 유형 |
|------|------|
| Q1: 선택할 때 고민이 되었나요? | 5점 척도 |
| Q2: 두 옵션 사이에 트레이드오프가 느껴졌나요? | 5점 척도 |
| Q3: 가장 고민되었던 카드는? | 선택형 |
| Q4: 적의 존재가 선택에 영향을 주었나요? | 5점 척도 |

## 8. 검증 기준

### 8.1 성공 기준
| 지표 | 기준 | 의미 |
|------|------|------|
| 평균 선택 시간 | ≥ 3초 | 멈춤 발생 |
| A/B 비율 | 40-60% | 균형 잡힌 트레이드오프 |
| 전환 횟수 | 평균 ≥ 1회 | 비교 행동 발생 |
| 설문 Q2 | 평균 ≥ 3.5점 | 주관적 충돌 인지 |
| 설문 Q4 | 평균 ≥ 3점 | 적이 맥락 제공 |

### 8.2 실패 기준 (Kill Criteria)
| 지표 | 기준 | 의미 |
|------|------|------|
| 평균 선택 시간 | < 2초 | 고민 없음 |
| A/B 비율 | > 75/25 | "정답" 존재 |
| 설문 Q1 | 평균 < 2점 | 고민 경험 없음 |

### 8.3 추가 분석
- **태그 유/무 비교**: 태그 있는 카드 vs 없는 카드 선택 시간 차이
- **HP 압박 분석**: 낮은 HP 상황에서 선택 패턴 변화
- **턴 진행 분석**: 후반 턴에서 고민 시간 변화

```
┌─────────────────────────────────────────┐
│  [적] ████████░░░░░░░░░░░░ 80/100 HP   │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │         카드 표시 영역           │   │
│  │     (A/B 선택 버튼 포함)         │   │
│  └─────────────────────────────────┘   │
│                                         │
│  [나] ████████████░░░░░░░░ 40/50 HP    │
│  [⚡] ████░░░░░░░░░░░░░░░░ 3 에너지     │
│                                         │
│  [공격] 버튼 (에너지 ≥ 3일 때 활성화)   │

```

### 9.2 최소 기능
- 단일 `index.html` 파일로 로컬 실행
- 카드 순차 표시 (6장)
- HP 바 표시 (적, 플레이어)
- 에너지 게이지 표시
- 공격 버튼
- A/B 선택 버튼
- 데이터 자동 기록

### 9.3 종료 처리
- 승리/패배 결과 표시
- 설문 표시
- CSV 다운로드 / JSON 복사 버튼

### 9.4 제약 사항
- 시간 제한 없음
- 선택 되돌리기 불가
- 카드 순서 고정 (무작위 X)
