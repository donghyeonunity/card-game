# 스플래시 씬 요구사항 문서

**작성일**: 2025-12-28
**대상**: Phase 1 프로토타입 (React) 및 Phase 2 본개발 (Unity) 공통 기반

---

## 1. 스플래시 씬 개요

### 1.1 씬 목적
- **브랜딩**: 개발자 정체성과 게임 타이틀을 명확히 전달
- **분위기 조성**: 전략적이고 냉철한 게임의 톤 앤 매너 설정
- **부드러운 진입**: 갑작스러운 화면 전환 없이 몰입감 있는 게임 시작
- **기술적 준비**: 리소스 프리로딩 시간 확보 (본개발)

### 1.2 진입 조건
- 게임 실행 직후 최초 진입
- 이전 씬: 없음 (앱 시작점)
- 다음 씬: 메인메뉴 씬

### 1.3 핵심 경험
플레이어는 스플래시 씬에서 다음을 경험해야 합니다:
- **깔끔한 첫인상**: 미니멀하고 세련된 비주얼로 게임의 정체성 인지
- **몰입 준비**: 어두운 배경과 절제된 애니메이션으로 집중 유도
- **선택권**: 원하면 스킵, 원치 않으면 자동 진행 (압박 없음)

### 1.4 디자인 철학
- **미니멀리즘**: 불필요한 요소 제거, 타이틀과 로고에만 집중
- **전략적 냉철함**: PRD의 "불완전한 정보 속 가치 충돌" 게임 특성을 시각적으로 암시
- **존중**: 플레이어 시간 존중 (스킵 가능)

## 2. 표시 요소 상세

### 2.1 화면 레이아웃

```
┌──────────────────────────────────────────────┐
│                                              │
│                                              │
│                                              │
│           [개발자/스튜디오 로고]              │
│              (페이드 인/아웃)                 │
│                                              │
│                   ↓                          │
│                                              │
│              [게임 타이틀]                    │
│           (메인 페이드 인/아웃)               │
│                                              │
│                                              │
│                                              │
│                                              │
│  [버전 정보]                                 │
│  v0.1.0                                      │
└──────────────────────────────────────────────┘

배경: 어두운 단색 (#0a0a0a)
```

### 2.2 개발자/스튜디오 로고

#### 표시 정보
- **내용**: 개발자 이름 또는 스튜디오 로고
- **위치**: 화면 중앙 상단 (중심에서 위쪽 20%)
- **크기**: 중간 크기 (화면 너비의 20~30%)

#### 스타일
| 속성 | 값 | 비고 |
|------|-----|------|
| 배경색 | 투명 | 배경과 자연스럽게 통합 |
| 색상 | #ffffff (흰색) 또는 연한 회색 (#cccccc) | 가독성 우선 |
| 폰트 | 고딕/산세리프 | 깔끔한 인상 |
| 정렬 | 가운데 | 균형감 |

#### 애니메이션
```
[0.0초] 완전 투명 (opacity: 0)
         ↓
[0.5초] 페이드 인 시작
         ↓
[1.5초] 완전 표시 (opacity: 1)
         ↓
[2.5초] 1초간 유지
         ↓
[3.5초] 페이드 아웃 시작
         ↓
[4.0초] 완전 투명 → 다음 요소로
```

### 2.3 게임 타이틀

#### 표시 정보
- **내용**: 게임 제목 (현재 미정 - 프로토타입에서는 "Card Game" 플레이스홀더 사용)
- **위치**: 화면 정중앙
- **크기**: 큰 크기 (화면 너비의 50~60%)

#### 스타일
| 속성 | 값 | 비고 |
|------|-----|------|
| 배경색 | 투명 | |
| 색상 | #ffffff (흰색) | 강한 대비 |
| 폰트 | 굵은 산세리프 또는 전용 타이틀 폰트 | 브랜드 아이덴티티 |
| 크기 | 48~64px (프로토타입) | 본개발에서 조정 |
| 정렬 | 가운데 | |
| 효과 | 약간의 그림자 또는 외곽선 (선택적) | 가독성 향상 |

#### 애니메이션
```
[4.0초] 완전 투명 (opacity: 0)
         ↓
[4.5초] 페이드 인 시작
         ↓
[5.5초] 완전 표시 (opacity: 1)
         ↓
[7.5초] 2초간 유지
         ↓
[8.0초] 페이드 아웃 시작 (스킵되지 않은 경우)
         ↓
[9.0초] 완전 투명 → 메인메뉴로 자동 전환
```

### 2.4 버전 정보

#### 표시 정보
- **내용**: "v0.1.0" (프로토타입), "v1.0.0" (본개발 출시 시)
- **위치**: 화면 좌측 하단 (여백 20px)
- **크기**: 작은 크기 (12~14px)

#### 스타일
| 속성 | 값 | 비고 |
|------|-----|------|
| 색상 | #666666 (어두운 회색) | 눈에 거슬리지 않게 |
| 폰트 | 모노스페이스 또는 산세리프 | 기술적 느낌 |
| 투명도 | 0.7 | 부드럽게 |

#### 표시 시점
- 게임 타이틀과 함께 표시 (4.5초부터)
- 씬 종료까지 유지

## 3. 전환 로직

### 3.1 자동 전환 타이밍

```
[ 씬 시작 ]
     ↓
[0.0 ~ 4.0초] 개발자 로고 시퀀스
     ↓
[4.0 ~ 9.0초] 게임 타이틀 시퀀스
     ↓
[9.0초] 자동 전환 → 메인메뉴 씬
```

**총 지속 시간**: 9초 (사용자 입력 없는 경우)

---

### 3.2 스킵 조건

#### 입력 감지 시점
- **최소 대기 시간**: 2초
  - 0~2초: 입력 무시 (로고가 최소한 보이도록)
  - 2초 이후: 입력 감지 시작

#### 스킵 가능 입력
| 입력 방식 | 감지 이벤트 | 비고 |
|----------|-----------|------|
| 마우스 클릭 | 화면 아무 곳이나 클릭 | 가장 직관적 |
| 키보드 | 아무 키나 (Enter, Space, ESC 등) | PC 환경 |
| 터치 | 화면 탭 (모바일 대비) | 향후 확장 |

#### 스킵 실행 시
```
[입력 감지]
     ↓
즉시 페이드 아웃 (0.5초)
     ↓
메인메뉴 씬으로 전환
```

**참고**: 화면에 "스킵 안내 문구"는 표시하지 않음 (스킵 기능 자체는 유지)

---

### 3.3 페이드 효과

#### 씬 진입 시
- 검은 화면에서 시작
- 0.5초간 페이드 인 (배경 + 첫 요소)

#### 씬 종료 시
- 현재 화면에서 0.5초간 페이드 아웃
- 완전 검은 화면 → 메인메뉴 씬 로드

#### 효과 상세
| 단계 | 효과 | 시간 |
|------|------|------|
| 진입 | 검은색 오버레이 opacity 1 → 0 | 0.5초 |
| 종료 | 검은색 오버레이 opacity 0 → 1 | 0.5초 |
| 이징 | ease-in-out | 부드러운 전환 |

## 5. UI/UX 명세

### 5.1 반응형 레이아웃

#### 화면 비율 대응
| 해상도 | 레이아웃 조정 |
|--------|-------------|
| 16:9 (일반) | 기본 레이아웃 |
| 21:9 (울트라와이드) | 좌우 여백 증가, 중앙 유지 |
| 4:3 (구형) | 타이틀 크기 약간 축소 |

#### 모바일 대응 (향후)
- 세로 모드: 타이틀 크기 조정
- 가로 모드: PC 레이아웃과 동일

---

### 5.2 접근성

#### 시각적 접근성
- **대비**: 흰색 텍스트 (#ffffff) vs 어두운 배경 (#0a0a0a) = 21:1 (WCAG AAA 통과)
- **크기**: 타이틀 최소 48px (가독성 확보)

#### 입력 접근성
- 마우스, 키보드 모두 지원
- 터치 입력 대비 (모바일 확장 시)

#### 시간 압박 없음
- 자동 전환이지만 충분한 시간 제공 (9초)
- 원하면 언제든 스킵 가능 (2초 후)

---

### 5.3 UX 원칙

#### 일관성
- 페이드 효과 타이밍 일관 (0.5초)
- 모든 애니메이션 ease-in-out 이징

#### 몰입 우선
- 불필요한 UI 없음 (진행바, 로딩 스피너, 스킵 안내 등 제거)
- 깔끔한 화면 (미니멀리즘)
