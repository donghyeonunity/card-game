# 배틀씬 요구사항 문서

**작성일**: 2025-12-28
**대상**: Phase 1 프로토타입 (React) 및 Phase 2 본개발 (Unity) 공통 기반

---

## 1. 배틀씬 개요

### 1.1 씬 목적
- 플레이어가 **즉각적 전술 선택**을 반복하며 가치 충돌을 체험하는 공간
- PRD의 **Layer 1 (반응 판단)** 을 구현하는 핵심 씬
- "선택지가 없으면 지루하다" 철학에 따라 **매 순간 의미 있는 선택** 제공

### 1.2 진입 조건
- 던전 진입 후 전투 노드 선택 시
- 이전 씬: 던전 맵 씬 (또는 프로토타입에서는 게임 시작)
- 다음 씬: 배틀 보상 씬 (승리 시) / 게임오버 씬 (패배 시)

### 1.3 핵심 경험
플레이어는 배틀 중 다음을 경험해야 합니다:
- **매 턴 의미 있는 선택**: 드로우한 5장 중 어떤 카드를 플레이할지
- **자원 관리 긴장감**: 에너지 수급 vs 즉시 공격의 딜레마
- **시간 압박**: 적의 반격으로 인한 턴 제한
- **가치 충돌 체감**: 단기↔장기, 안정↔변동 등의 트레이드오프

## 2. 배틀씬 내 선택 포인트

### 2.1 핵심 선택: 카드 플레이 선택 (매 턴)

#### 발생 시점
- 매 턴 시작 시 카드 5장 드로우 후
- 플레이어가 턴을 종료할 때까지 반복

#### 선택지 구성
```
드로우: 5장 카드 제시
      ↓
플레이어 선택:
  - 카드 1~5장을 순서대로 플레이
  - 일부 카드는 남겨두고 턴 종료 가능
  - 남은 카드는 버려짐 (다음 턴에 새로 드로우)
```

#### 구체적 예시
```
[ 턴 시작 ]
현재 상태: HP 40/50, 에너지 2, 적 HP 80/100

드로우된 카드 5장:
┌────────┬────────┬────────┬────────┬────────┐
│ 1. 급속 │ 2. 정밀 │ 3. 효율 │ 4. 방어 │ 5. 강타 │
│ 충전   │ 타격   │ 충전   │ 태세   │        │
│ +3에너지│ 3에너지│ 2턴간  │ 반격   │ 3에너지│
│        │ →20딜 │ +2에너지│ 무효화 │ →10~40 │
└────────┴────────┴────────┴────────┴────────┘

플레이어 고민:
"지금 에너지가 2인데... 어떻게 하지?"

옵션 A: [1번 급속 충전] → [2번 정밀 타격] 플레이
  → 에너지 2 → 5 → 2
  → 적에게 20 데미지 (60/100)
  → 빠르게 처리하는 안정적 루트

옵션 B: [3번 효율 충전] → [5번 강타] 플레이
  → 에너지 2 → 2 (이번 턴), 다음 2턴 +2씩
  → 강타 실패 시 적 HP 70 남음
  → 도박이지만 크리티컬 시 대박

옵션 C: [4번 방어 태세]만 플레이하고 턴 종료
  → 이번 턴 반격 -10 방어
  → 다음 턴을 위한 여유 확보
  → 하지만 적 HP는 그대로...
```

#### 플레이어 예상 감정
- **선택 전**: "5장 중 뭘 써야 최선일까?" (전술적 고민)
- **에너지 부족 시**: "에너지 카드를 먼저 써야 하나, 아니면 다음 턴으로 미룰까?" (자원 압박)
- **위험 카드 등장 시**: "강타를 쓰면 끝낼 수도 있는데... 실패하면?" (유혹과 불안)
- **선택 후**: "이게 맞았나?" (즉각 피드백 대기)
- **결과 확인 후**: "역시 내 선택이 옳았어!" (만족) 또는 "아... 다른 걸 할 걸" (후회)

#### 게임에 미치는 영향
- **즉시 영향**: 에너지 증감, 적 HP 변화, 반격 방어 등
- **누적 영향**: 매 턴 선택이 쌓여 승리/패배로 이어짐
- **전략 형성**: "나는 안정형이다 / 공격형이다" 플레이 스타일 드러남

---

### 2.2 보조 선택: 공격 타이밍 (턴 내)

#### 발생 시점
- 에너지가 3 이상일 때
- 플레이어가 원하는 시점에 공격 버튼 클릭

#### 선택지 구성
```
에너지 ≥ 3일 때:
  - 지금 공격 (에너지 3 소모 → 20 데미지)
  - 에너지 더 모아서 다음 턴에 공격
  - 카드 효과로 강화된 공격 사용 (카드가 있을 경우)
```

#### 플레이어 예상 감정
- **선택권 느낌**: "내가 타이밍을 정할 수 있다" (통제감)
- **기회비용 고민**: "지금 쓸까, 모아서 한방에 날릴까?" (최적화 욕구)
- **압박감** (HP 낮을 때): "지금 안 치면 죽는다!" (긴박함)

---

### 2.3 전략적 선택: 턴 종료 시점

#### 발생 시점
- 카드를 일부 또는 전부 플레이한 후
- "턴 종료" 버튼 클릭 시

#### 선택지 구성
```
카드 플레이 중:
  - 남은 카드를 더 플레이할까?
  - 아니면 지금 턴을 종료할까?

턴 종료 시:
  - 남은 카드는 모두 버려짐
  - 적의 반격 받음 (-10 HP)
  - 다음 턴 시작 (새로 5장 드로우)
```

#### 플레이어 예상 감정
- **손실 회피**: "카드를 버리는 게 아깝다" (낭비 불안)
- **속도감**: "빨리 넘어가고 싶은데 카드가 애매하다" (효율 고민)
- **운영 컨트롤**: "내 페이스로 게임을 조절한다" (통제감)

### 3.2 턴별 상세 흐름

#### 턴 1 예시
```
[ 턴 시작 ] (남은 턴: 5)
  플레이어: HP 50/50, 에너지 0
  적: HP 100/100

  ↓ 카드 5장 드로우

[ 카드 플레이 ]
  플레이어 선택:
    1) "급속 충전" 카드 플레이 → 에너지 +3
    2) "정밀 타격" 카드 플레이 → 에너지 3 소모, 적 -20 HP
    3) 턴 종료 선택

  결과:
    플레이어: HP 50, 에너지 0
    적: HP 80/100

  ↓ 턴 종료

[ 적 반격 ]
  플레이어 HP -10 → 40/50

[ 다음 턴으로 ] (남은 턴: 4)
```

#### 턴 5 예시 (최종 턴 압박 상황)
```
[ 턴 시작 ] (남은 턴: 1 - 마지막 턴!)
  플레이어: HP 10/50, 에너지 4
  적: HP 25/100

  "이번 턴에 못 끝내면 패배!"

  ↓ 카드 5장 드로우

[ 카드 플레이 ]
  드로우: [강타 3에너지], [회복 +15HP], [에너지+2], ...

  플레이어 고민:
    A: 강타 사용 (10~40 무작위) → 운 좋으면 승리
    B: 회복 먼저 (+15 HP) → 턴 제한으로 패배
    C: 에너지 더 모아서 확정 딜 넣기 → 시간 없음!

  감정: "도박할 수밖에 없다!" (극한의 압박)
```

## 4. 유닛 시스템

### 4.1 플레이어

| 속성 | 초기값 | 설명 |
|------|--------|------|
| **HP** | 50 | 생존 조건, 0 이하 시 패배 |
| **에너지** | 0 | 카드/공격에 사용되는 자원 |
| **최대 HP** | 50 | 회복 상한선 |

#### 플레이어 행동
- 카드 플레이
- 공격 (에너지 3 소모 → 20 데미지)
- 턴 종료

### 4.2 적

| 속성 | 초기값 | 설명 |
|------|--------|------|
| **HP** | 100 | 처치 목표, 0 이하 시 승리 |
| **반격** | 10/턴 | 매 턴 종료 시 플레이어에게 자동 데미지 |

#### 적 행동
- 턴 종료 시 자동 반격 (-10 HP)
- (향후 확장) 특정 턴에 특수 패턴

## 5. 카드 시스템

### 5.1 카드 구성
- 총 카드 풀: 20~30장 (프로토타입에서는 12장)
- 매 턴 드로우: 5장 무작위
- 플레이 후: 버려짐 (다음 턴에 다시 풀에서 드로우 가능)

### 5.2 카드 타입 (기능별)

#### A. 에너지 카드
- 목적: 자원 수급
- 예시:
  - "급속 충전": 즉시 +3 에너지
  - "효율 충전": 2턴간 턴 시작 시 +2 에너지

#### B. 공격 카드
- 목적: 적 HP 감소
- 예시:
  - "정밀 타격": 에너지 3 → 확정 20 데미지
  - "강타": 에너지 3 → 10~40 무작위 데미지

#### C. 방어 카드
- 목적: 생존력 증가
- 예시:
  - "방어 태세": 이번 턴 반격 무효화
  - "즉시 회복": HP +15

#### D. 전략 카드
- 목적: 복합 효과, 트레이드오프 제시
- 예시:
  - "집중 투자": 보유 에너지 전부 소모 → 소모량 × 8 데미지
  - "위험 감수": 반격 받음 + 에너지 +2 보너스

### 5.3 카드 정보 표시

```
┌─────────────────────────────────┐
│  [카드명]                        │
├─────────────────────────────────┤
│  효과 설명:                      │
│  - 에너지 +3 즉시 획득            │
│  - ...                          │
│                                 │
│  [태그: 단기↑]                   │
│  (선택적, 일부 카드만)            │
└─────────────────────────────────┘
```

## 6. 에너지 시스템

### 6.1 에너지 획득
- 카드 효과로만 획득 (자동 충전 없음)
- 예시:
  - "급속 충전" 카드 → +3
  - "효율 충전" 카드 → 2턴간 +2씩
  - "위험 감수" 카드 → +2 (반격 대가)

### 6.2 에너지 소모
- 기본 공격: 3 에너지
- 강화 공격 카드: 카드별 상이 (3~5)
- 일부 전략 카드: 보유 에너지 전부 소모

### 6.3 에너지 이월
- 턴 종료 시 **에너지는 유지됨** (소멸 안 됨)
- 누적 가능 (상한선: 10~15, 프로토타입에서는 무제한)

---

## 7. 승리/패배 조건

### 7.1 승리 조건
- **적 HP ≤ 0**
- 즉시 배틀 종료 → 보상 씬으로 이동

### 7.2 패배 조건
- **플레이어 HP ≤ 0**
- **턴 제한 초과** (5턴 내 처치 실패)
- 즉시 배틀 종료 → 게임오버 씬으로 이동

### 7.3 턴 제한 (명시)
- **기본 턴 제한: 5턴**
- 화면에 "남은 턴: X" 형태로 명시
- 방어 카드 사용 시 1~2턴 연장 가능 (향후 확장)

### 7.4 동시 처치
- 플레이어 공격으로 적 HP 0 도달 후 반격으로 플레이어 HP 0 도달 시
- **플레이어 승리로 처리** (공격이 먼저 해결됨)

| 적 HP 바 | 현재/최대 HP 표시 | 데미지 받으면 애니메이션 |
| 적 이미지 | 시각적 존재감 | 공격 시 히트 이펙트 |
| 반격 표시 | "턴 종료 시 -10" 경고 | 항상 표시 (압박감) |

#### C. 카드 영역
| 요소 | 설명 | 동작 |
|------|------|------|
| 카드 5장 | 드로우된 카드 표시 | 클릭 시 플레이 |
| 카드 상세 정보 | 호버 시 효과 상세 표시 | 툴팁/확대 |
| 플레이된 카드 표시 | 이번 턴에 사용한 카드 | 시각적 구분 (흐리게 등) |
| 남은 카드 강조 | 아직 안 쓴 카드 하이라이트 | 선택 유도 |

#### D. 플레이어 정보 영역
| 요소 | 설명 | 동작 |
|------|------|------|
| HP 바 | 현재/최대 HP | 반격 시 감소 애니메이션 |
| 에너지 게이지 | 현재 에너지 (숫자 + 바) | 증감 시 애니메이션 |
| 공격 버튼 | 에너지 ≥ 3 시 활성화 | 클릭 시 공격 실행 |
| 턴 종료 버튼 | 항상 활성화 | 클릭 시 턴 종료 확인 |

#### E. 피드백 요소
| 요소 | 설명 | 동작 |
|------|------|------|
| 데미지 숫자 | 데미지 발생 시 팝업 | "-20" 빨간색 표시 |
| 에너지 변화 표시 | "+3" 초록색 팝업 | 카드 효과 시 |
| 효과 지속 표시 | "2턴간 +2 에너지" 아이콘 | 플레이어 정보 옆 |

### 8.3 정보 표시 우선순위

#### 핵심 정보 (항상 표시)
- **남은 턴 수** (명시)
- 플레이어 HP / 에너지
- 적 HP
- 드로우된 카드 5장

#### 보조 정보 (필요 시 표시)
- 지속 효과 아이콘
- 반격 경고

#### 숨김 가능 (호버/클릭 시)
- 카드 상세 설명
- 전투 로그
- 통계 (총 데미지 등)

### 8.4 UX 원칙

#### 즉각 피드백
- 모든 행동에 **0.3초 이내** 시각/청각 피드백
- 카드 클릭 → 효과 애니메이션 → 상태 변화 순서 명확히

#### 실수 방지
- 턴 종료 시 "남은 카드 버려짐" 경고 (첫 1~2턴만)
- 공격 버튼 활성화 시 시각적 강조

#### 선택 명확성
- 현재 선택 가능한 행동 하이라이트
- 불가능한 행동은 비활성화 + 이유 표시

#### 압박감 조성
- **남은 턴 수 명시** (상단에 크게 표시)
- 적 반격 표시 (빨간색 경고)
- HP 낮을 때 (30% 이하) HP 바 깜빡임
- 마지막 턴에 화면 전체 경고 효과

## 9. 데이터 구조

### 9.1 플레이어 상태 (Player State)
```typescript
interface PlayerState {
  hp: number;           // 현재 HP
  maxHp: number;        // 최대 HP
  energy: number;       // 현재 에너지
  maxEnergy?: number;   // 에너지 상한 (선택적)
  effects: Effect[];    // 지속 효과 배열
}
```

### 9.2 적 상태 (Enemy State)
```typescript
interface EnemyState {
  hp: number;           // 현재 HP
  maxHp: number;        // 최대 HP
  attackPower: number;  // 반격 데미지 (기본 10)
  effects: Effect[];    // 지속 효과 (향후 확장)
}
```

### 9.3 카드 데이터 (Card)
```typescript
interface Card {
  id: string;                // 고유 ID
  name: string;              // 카드명
  description: string;       // 효과 설명
  energyCost?: number;       // 에너지 비용 (있으면)
  effects: CardEffect[];     // 효과 배열
  tags?: ValueTag[];         // 가치축 태그 (선택적)
}

interface CardEffect {
  type: 'energy' | 'damage' | 'heal' | 'defend' | 'buff' | 'debuff';
  value: number | Range;     // 수치 (또는 범위)
  target: 'player' | 'enemy';
  duration?: number;         // 지속 턴 (있으면)
}

interface ValueTag {
  axis: '단기↔장기' | '안정↔변동' | '효율↔확장' | '통제↔자유';
  direction: '↑' | '↓';
}
```

### 9.4 전투 상태 (Battle State)
```typescript
interface BattleState {
  turn: number;              // 현재 턴
  maxTurn: number;           // 최대 턴 (기본 5)
  phase: 'draw' | 'play' | 'end' | 'result';
  player: PlayerState;
  enemy: EnemyState;
  hand: Card[];              // 현재 핸드 (5장)
  playedCards: Card[];       // 이번 턴에 플레이한 카드
  deck: Card[];              // 카드 풀
  battleLog: LogEntry[];     // 전투 기록
}
```

### 9.5 효과 (Effect)
```typescript
interface Effect {
  id: string;
  name: string;              // "효율 충전"
  type: 'energy_per_turn' | 'defense' | 'attack_buff';
  value: number;
  remainingTurns: number;    // 남은 지속 턴
}
```

## 11. PRD 가치 충돌 모델 구현

### 11.1 카드별 가치 축 매핑

| 카드 타입 | 주요 충돌 | 예시 |
|----------|----------|------|
| 에너지 수급 | 단기 ↔ 장기 | 즉시 +3 vs 2턴간 +2씩 |
| 공격 방식 | 안정 ↔ 변동 | 확정 20 vs 10~40 무작위 |
| 자원 배분 | 효율 ↔ 확장 | 전부 소모 고데미지 vs 일부만 쓰고 유지 |
| 방어 선택 | 통제 ↔ 자유 | 반격 무효화 + 제약 vs 반격 받고 자유 |

### 11.2 Layer 1 (반응 판단) 구현
- **빈도**: 매 턴 5장 카드 선택 → 매우 잦음
- **무게**: 개별 카드는 가벼운 선택
- **특징**:
  - 5장을 보고 즉각 반응
  - 제한된 정보 (다음 드로우 알 수 없음)
  - 결과는 빠르게 나타남 (즉시 효과)
  - 실수 가능성 존재 (잘못된 카드 선택)

### 11.3 누적 → Layer 2 연결 (향후 구현)
- 배틀씬 자체는 Layer 1 중심
- Layer 2는 **배틀 보상 씬**에서 구현:
  - 배틀에서의 플레이 스타일 요약
  - "당신은 공격형으로 플레이했습니다" 피드백
  - 보상 3개 중 1개 선택 (덱 편집, 유물 획득 등)
  - "지금까지 방향이 맞았나?" 재평가

## 12. 사운드 및 이펙트

### 12.1 BGM
- 배틀 중: 긴장감 있는 전투 BGM
- 마지막 턴: 더 긴박한 BGM으로 전환
- 승리 시: 짧은 승리 팡파레
- 패배 시: 어두운 패배 사운드

### 12.2 효과음
| 상황 | 사운드 |
|------|--------|
| 카드 드로우 | 카드 섞는 소리 |
| 카드 플레이 | 카드 놓는 소리 |
| 에너지 획득 | 밝은 "챙그랑" |
| 공격 성공 | 타격음 + 적 신음 |
| 적 반격 | 어두운 타격음 |
| HP 감소 | 낮은 "쿵" |
| 턴 종료 | 부드러운 전환음 |
| 마지막 턴 경고 | 경고음 |

### 12.3 시각 효과
| 상황 | 이펙트 |
|------|--------|
| 카드 플레이 | 카드 → 효과 부위로 날아가는 애니메이션 |
| 데미지 | 빨간 숫자 팝업 + 적 흔들림 |
| 에너지 획득 | 초록 빛 파티클 |
| 반격 | 적 → 플레이어 방향 충격파 |
| 마지막 턴 | 화면 테두리 붉은색 깜빡임 |
| 승리 | 적 사라짐 + 빛나는 효과 |
| 패배 | 화면 어두워짐 |

## 14. 향후 논의 필요 사항

### 14.1 카드 밸런스
- 기본 공격 20 데미지 (5방이면 적 처치)
- 적 반격 10 데미지 (5턴이면 플레이어 사망)
- **제안**: 프로토타입 테스트 후 조정

### 14.2 카드 드로우 방식
- A안: 덱에서 완전 무작위 5장
- B안: "에너지 카드 최소 1장 보장" 같은 규칙
- **제안**: 프로토타입에서는 A안 (완전 무작위)

### 14.3 덱 크기
- 프로토타입: 12~15장
- 본개발: 20~30장 (던전 진행하며 증가)

### 14.4 지속 효과 표시
- A안: 플레이어 정보 옆 (HP/에너지 바 옆)
- B안: 화면 상단 별도 영역
- **제안**: 프로토타입에서 A안 테스트

### 14.5 적의 다양성
- 프로토타입: 동일 패턴 (반격 10)
- 본개발: 적 타입 3~5종

### 14.6 카드 플레이 순서 제약
- **제안**: 자유 순서 (1→5→2 가능)

### 14.7 Layer 2 배틀 내 통합 여부
- 현재 설계: 배틀씬 = Layer 1, 보상 씬 = Layer 2
- 대안: 배틀 중 특정 턴(3턴)에 "전환점 이벤트"
- **제안**: 프로토타입에서 Layer 1만, 추후 고려

### 14.8 패배 시 재시작
- 프로토타입: 게임오버 → 처음부터
- 본개발: 던전 입구부터 재시작 (로그라이크)
