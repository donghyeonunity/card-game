# PRD - 에너지 기반 카드 게임

> **문서 목적**: Unity 본개발을 위한 프로젝트 정의서

---

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **장르** | 로그라이크 덱빌딩 카드 게임 |
| **플랫폼** | Steam |
| **전투 방식** | 턴제 |
| **런 플레이타임** | 20~30분 |
| **타겟 유저** | 캐주얼 ~ 코어 |

### 타겟 유저 전략

| 설계 | 진입은 쉽게, 마스터는 어렵게 |
|------|------|
| **왜?** | 낮은 진입장벽으로 넓은 유저 확보, 높은 난이도 옵션으로 코어 팬 만족. "배우기 쉽고 마스터하기 어려운" 게임. |

## 2. 핵심 철학

> 상세 내용은 `docs/00. project/PRD.md` 참조

### Game Pillar
플레이어가 **불완전한 정보 속에서 여러 가치 축이 동시에 충돌하는 상황**에 즉각 반응하며, 그 순간적인 판단들이 누적되어 중간 무게의 결정을 다시 요구받는 게임.

### Core Experience
**반응 → 누적 → 재평가**의 반복에서 몰입 발생

### 가치 충돌 모델 (4개 축)
| 축 | 의미 |
|----|------|
| 단기 ↔ 장기 | 즉시 효과 vs 턴 투자 후 큰 효과 |
| 안정 ↔ 변동 | 확정 결과 vs 고위험 고보상 |
| 효율 ↔ 확장 | 자원 최적화 vs 선택지 확보 |
| 통제 ↔ 자유 | 행동 제한 + 안전 vs 위험 감수 + 자유 |

## 3. 게임 시스템

### 3.1 캐릭터 설정

| 설계 | 질문/선택지 기반 → 덱 + 능력치 구성 |
|------|------|
| **왜?** | 플레이어가 "나만의 캐릭터"를 만들었다는 소유감과 애착을 느끼게 하기 위해. TRPG처럼 캐릭터 생성 과정 자체가 몰입의 시작점. |

---

### 3.2 던전 구조

| 설계 | 챕터/구역별 진행 (구역마다 보스) |
|------|------|
| **왜?** | 중간 성취감("드디어 클리어!")과 기대감("다음은 뭐지?")을 주기적으로 제공. 긴 여정에서도 작은 승리를 맛보게 함. |

| 설계 | 길목에서 플레이어가 경로 선택 |
|------|------|
| **왜?** | 선택의 주도권을 플레이어에게 부여. "내가 이 던전을 탐험하고 있다"는 능동적 경험 제공. |

---

### 3.3 전투 시스템

#### 자원 시스템
| 설계 | 자원 제약 시스템 |
|------|------|
| **왜?** | 제약이 있어야 고민이 생기고, 고민이 있어야 몰입이 된다. 무한한 자원은 무의미한 선택을 만든다. |

#### 카드 드로우
| 설계 | 매 턴 새로운 카드 드로우 |
|------|------|
| **왜?** | 매 턴 새로운 환경(핸드)을 마주하게 하여 "이번 턴엔 어떻게 해야 하지?"라는 전략 재수립 유도. 반복이 아닌 적응의 재미. |

#### 적 정보 공개
| 설계 | 적 행동 완전 공개 (다음 턴 행동 표시) |
|------|------|
| **왜?** | 정보를 바탕으로 계산하고 판단하는 **전략적 쾌감**. "적이 20 공격하니까 방어 먼저" 같은 퍼즐 풀이의 만족감. |

#### 전투 템포
| 설계 | 초반 빠른 전투(3~5턴) → 후반 중간 전투(6~10턴) |
|------|------|
| **왜?** | 초반엔 빠른 피드백으로 리듬감, 후반엔 복잡한 전투로 성장한 덱의 힘 체감. 난이도와 깊이가 함께 성장. |

---

### 3.4 덱 관리

| 설계 | 카드 제거 + 업그레이드 + 빌딩 모두 지원 |
|------|------|
| **왜?** | 다양한 전략 스타일 허용. 날씬한 콤보덱, 신중한 밸런스덱, 강화 중심덱 모두 가능. 플레이어마다 자신만의 방식으로 덱 완성. |

---

### 3.5 보상 시스템

| 보상 | 내용 | 플레이어 감정 |
|------|------|---------------|
| **카드** | 전투 후 3장 중 1장 선택 | "이것도 좋고 저것도 좋은데..." 선택의 고민 |
| **골드** | 상점에서 사용 | 자원 관리의 압박 |
| **보물** | 지속적 패시브 효과 아이템 | 빌드 강화의 만족감 |

---

### 3.6 상점 시스템

| 설계 | 한정된 골드 + 미확인 아이템 |
|------|------|
| **왜?** | 한정 골드는 "둘 다 갖고 싶은데..."라는 **선택의 고통**을, 미확인 아이템은 "뭐가 나올까?"라는 **도박의 기대감**을 제공. 안전한 선택과 모험 사이의 긴장. |

## 4. 세계관 & 아트

### 4.1 아트 스타일

| 설계 | 클래식 판타지 (밝고 선명한 색감) |
|------|------|
| **왜?** | 모험의 설렘과 친근함. 진입 장벽을 낮추고, 따뜻하면서도 모험심을 자극하는 분위기. |

### 4.2 스토리 방향

| 설계 | 판타지 세계 + 현실 고민 함축 |
|------|------|
| **왜?** | 익숙한 판타지로 진입 장벽을 낮추면서, 현실적 고민을 담아 "이건 내 이야기 같다"는 공감대 형성. 표면은 판타지, 내면은 공감. |

### 4.3 몬스터 등급

| 등급 | 역할 | 플레이어 감정 |
|------|------|---------------|
| **일반** | 기본 전투, 리듬 형성 | 가벼운 긴장감, 진행의 흐름 |
| **엘리트** | 도전적 전투 | 집중과 긴장 상승 |
| **보스** | 구역 마무리 | 클라이맥스, 성취감 |

| 설계 | 일반 → 엘리트 → 보스 긴장도 파도 |
|------|------|
| **왜?** | 긴장과 이완의 완급 조절. 계속 긴장만 하면 지치고, 이완만 하면 지루함. 감정의 파도를 설계. |

## 5. 기술 스택

### 5.1 본개발 (Unity)

| 항목 | 선택 | 비고 |
|------|------|------|
| **엔진** | Unity 6 (LTS) | 최신 LTS, 장기 지원 |
| **렌더링** | URP | 2D 카드 게임에 적합, 가볍고 최적화 용이 |
| **UI 시스템** | 미정 | uGUI vs UI Toolkit 검토 필요 |
| **아키텍처** | ECS | 데이터 지향 설계, 성능/확장성 우선 |

### 에셋/플러그인

| 에셋 | 용도 |
|------|------|
| **DOTween** | 카드 이동, UI 애니메이션 |
| **Addressables** | 에셋 번들 관리, 메모리 최적화 |
| **Localization** | 다국어 지원 |
| **Steamworks.NET** | Steam 연동 (업적, 클라우드 저장, 리더보드) |

---

### 5.2 프로토타입 (웹)

| 항목 | 선택 |
|------|------|
| **프레임워크** | React 19 + TypeScript 5.9 (strict mode) |
| **빌드** | Vite 7.2 |
| **상태 관리** | Zustand 5.0 |
| **스타일링** | Tailwind CSS 4.1 |
| **애니메이션** | Framer Motion 12 |

| 설계 | 웹 기반 프로토타입 |
|------|------|
| **왜?** | 빠른 반복 개발로 아이디어를 신속하게 구현. Unity 본개발 전에 핵심 재미를 검증. 잘못된 방향에 큰 비용을 투자하기 전에 저비용으로 테스트. |

> 프로토타입 상세: `docs/10. prototype/Prototype-Game.md` 참조
